- В штатных сценариях сторож не срабатывает: полный прогон, продолжение после стоп-точки
  ниш и ↺ на любом модуле — зависимости всегда либо готовы, либо в плане.

оставались выдуманными. Проверить именно это в следующем прогоне: URL персон в M4

## [КРИТИЧНЫЙ БАГ, НАЙДЕН И ПОЧИНЕН] agent_package.json терял все ниши кроме одной
Обнаружено на реальном тестовом прогоне (2 ниши: «Интернет-магазины» + «Digital-агентства»,
файл CA_Ловец-Лидов_agent_package.json) — schema_version был 1, project.selectedNiche
содержал обе ниши, но modules{} (M2–M7) — данные только ПРО ОДНУ. Причина: buildAgentPackage
писал `modules[r.id] = {...}` по одному и тому же id ('M2','M3'...) для каждой ниши — объект
держит только последнюю записанную нишу, остальные молча перезаписывались.
Фикс (schema_version → 2):
- buildModuleEntry(r)/pickTable() вынесены в переиспользуемые функции.
- buildAgentPackage: по-нишевые результаты (M2–M7) группируются в
  research_by_niche[<имя ниши>] = { key_data, modules }; глобальные (M1_2) — в global_modules;
  добавлено verbatim-поле selected_niches (какие ниши реально исследованы в этом прогоне).
- key_data.segments (проектный уровень, не по нише) остался на верхнем уровне пакета.
- КОНТРАКТ-agent_package.md переписан под v2 (research_by_niche, global_modules,
  selected_niches; убрано упоминание lint_warnings — линтер уже снят коммитом 3d8b877,
  документ был устаревшим).
Заодно (тот же баг косвенно бил и по HTML-отчёту): generateHTMLReport строил графики M2/M4/M7
через `.find(r=>r.id===...)` — брал только ПЕРВУЮ по порядку нишу для графика (карта
конкурентов, осознанность, семантика). Сами текстовые секции по нишам не терялись (`.map()`
по всем результатам), но шли без подписи, какой нише принадлежат, и вперемешку при ↺
одной ниши (перегенерированная ниша падает в конец списка). Почищено:
- Секции модулей отсортированы как в run(): глобальные → ниша за нишей → порядок модулей.
- Перед первым модулем каждой новой ниши — разделитель «Ниша: …» (только если ниш > 1).
- Кнопка ↓ скачивания секции добавляет нишу в имя файла.
НЕ ПОЧИНЕНО (осталось на потом): графики M2 (карта конкурентов), M4 (осознанность),
M7 (семантика) в HTML-отчёте по-прежнему рисуются только для ОДНОЙ (первой) ниши —
чтобы показать по графику на каждую нишу, generateHTMLReport надо переписать на цикл
по нишам с уникальными canvas id на нишу. Не тронуто в этом заходе: правка графиков
крупнее, а проверить её можно только новым прогоном (лимиты Tavily/OpenAI ограничены).
Правки НЕ задеплоены — нужен `git push`, затем при необходимости пустой коммит-пинок,
если Vercel не подхватит сам (см. «Деплой (22 июля)» выше).
