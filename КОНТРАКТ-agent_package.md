# Контракт `agent_package.json` — CA Research → контент-машина

**Версия схемы:** 1 · **Обновлено:** 22 июля 2026

Файл выгружается кнопкой в CA Research (`CA_<проект>_agent_package.json`). Это структурированный
результат исследования для потребителя (контент-машина LeadYup). Здесь описаны поля, чтобы на
стороне потребителя не парсить markdown заново.

## Базовые правила

- **Язык.** Все данные — на **языке исследования** (`research_language`), в подлинных
  формулировках аудитории, без перевода. Перевод туда-обратно убивает живые цитаты (VoC),
  поэтому его тут принципиально нет. Если нужен перевод для чтения — это делается на стороне
  потребителя как отдельный слой, а в работу (темники, скрипты) идёт оригинал.
- **Версионирование.** Завязывайтесь на `schema_version`. Значение может расти при добавлении
  модулей M8–M10; существующие поля в рамках версии 1 не переименовываются.
- **Пустые данные — это данные.** Если таблица пустая или значение «не замерено»/«гипотеза,
  не проверено» — так и передаётся. Не додумывайте: отсутствие цифры значит, что её не замеряли.

## Верхний уровень

| Поле | Тип | Что это |
|---|---|---|
| `schema_version` | number | Версия контракта (сейчас 1). |
| `generated_at` | ISO-строка | Когда выгружен пакет. |
| `research_language` | строка | Язык, на котором созданы и хранятся данные (напр. `Russian`). |
| `project` | объект | Бриф: название, гео, рынок, ниша, выбранные ниши/услуги и т.д. |
| `niches` | массив | Ниши со стоп-точки M1.2 (уже структурированы, см. ниже). |
| `key_data` | объект | Быстрые ссылки на самое нужное (сегменты, боли, персоны, кластеры, гипотезы). |
| `modules` | объект | Все модули: таблицы объектами + полный markdown. |
| `executive_summary` | строка | Итоговый Decision Brief (markdown), если сгенерирован. |
| `price_layers` | объект | `{ available: [...], selected: [...] }` — ценовые слои, если использовались. |

## `niches[]` (со стоп-точки выбора ниши)

Каждый элемент:

| Поле | Тип | Пример |
|---|---|---|
| `name` | строка | «интернет-магазины» |
| `demand`, `competition`, `economics`, `fit` | number (1–5) | оценки |
| `score` | number | сумма |
| `verdict` | строка | `Go` / `Maybe` / `No-Go` |
| `why` | строка | одно предложение обоснования |
| `recommended` | bool | ровно у одной ниши `true` — рекомендация ИИ (человек мог выбрать другую) |

## `key_data`

Каждое поле — массив объектов (строки соответствующей таблицы модуля, ключи = заголовки колонок
на языке исследования). Состав колонок может слегка меняться между прогонами — читайте по имени
колонки, не по индексу.

| Поле | Источник | Что внутри |
|---|---|---|
| `segments` | M1.2 / BLOCK 04 | сегменты ЦА, платёжеспособность, квалификация |
| `competitors` | M2 / BLOCK 06 | карта конкурентов: оффер, цены, сильные/слабые стороны, gap |
| `voc` | M3 / BLOCK 07 | боли + дословные цитаты с URL, частотность, сегмент |
| `personas` | M4 / BLOCK 09 | персоны ЛПР: JTBD, боли, возражения, триггеры, каналы, цитаты |
| `jtbd` | M4 / BLOCK 10 | карта Jobs-to-be-Done |
| `hypotheses` | M5 / BLOCK 16 | гипотезы для A/B: формулировка, метрика, ожидаемый эффект |
| `seo_clusters` | M7 / SEO-02 | семантические кластеры (частотность — только с источником, иначе «не замерено») |

Если модуль не выполнялся — соответствующий массив пустой.

## `modules[<id>]`

Ключи — `M1_2`, `M2`, `M3`, `M4`, `M5`, `M6`, `M7` (и позже `M8`–`M10`). Каждый:

| Поле | Тип | Что это |
|---|---|---|
| `title` | строка | Название модуля на языке специалиста. |
| `generated_at` | ISO/`null` | Когда сгенерирован модуль. |
| `tables` | объект | `{ "<заголовок блока>": [ {строки}… ] }` — все таблицы модуля. |
| `markdown` | строка | Полный текст модуля без обрезки — на случай, если таблиц недостаточно. |
| `lint_warnings` | массив строк | *(опционально)* подозрение на непереведённые иноязычные вставки. |

Ключи в `tables` — это заголовки блоков как в отчёте (например `BLOCK 07 — Voice of Customer…`,
`BLOCK 04_2 — Niche Prioritization Matrix`, `SEO-02 — Semantic Core`). При совпадении заголовков
второй получает суффикс ` (N)`.

## Что важно помнить потребителю

- **Проверяемость.** У цитат VoC и у конкурентов есть URL — их можно и нужно открывать.
  Ссылки `[n]` в тексте M1.2/M7 расшифровываются в блоке источников того же модуля.
- **Числа с оговоркой.** Частотности SEO, прогнозы трафика, «рост до X%» помечаются как
  гипотеза/не замерено. Не переносите их в рекламные тексты как факт.
- **Персоны и цитаты связаны.** URL у персоны (M4) должен встречаться в таблице VoC (M3) —
  это признак того, что персона построена на реальных данных, а не выдумана.
